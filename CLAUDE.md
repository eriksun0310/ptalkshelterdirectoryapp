# Claude Code 專案指引

## 語言設定

- 全部使用**繁體中文**回覆

## 溝通模式

當我說「**分析和我討論**」或「**先不改 code**」時：
- 不要直接修改程式碼
- 列出可行方案，分析優缺點
- 提出最優解建議
- 等待我確認後再動手

## 程式碼品質要求

- **嚴禁**快速解決、低效、醜陋的寫法
- 優先考慮可維護性和可讀性
- 遵循專案既有的程式碼風格
- 不過度工程化，保持簡潔

## 設計風格

- 極簡風格：黑、白、灰 + 單一主色（藍 `#4A90E2`）
- 策略性使用主色標記重點（標題、數字、互動元素）
- 避免過多顏色造成視覺混亂
